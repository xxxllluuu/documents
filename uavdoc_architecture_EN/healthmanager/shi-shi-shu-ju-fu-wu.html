Monitoring Data Server

Monitoring Data Server

Monitoring Data Server consumes the monitoring data in RocketMQ. Below is the detailed workflow:

Monitoring Data Server consumes the monitoring data in RocketMQ delivered by MA, stores the data in OpenTSDB, and caches the data within 5 minutes in Redis. Monitoring Data Server also redelivers the monitoring data to RocketMQ and the redelivered data is further consumed by the Runtime Notification Server for notification information calculation. In addition, the Monitoring Data Server exposes the query interfaces for monitoring data query within 5 minutes via http://ip:port/hm/cache/q and for earlier data query via http://ip:port/hm/query by external applications.
